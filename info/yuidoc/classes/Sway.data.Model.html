Sway.data.Model Class

Defined in: src/data/activerecord.js:293

Use the Model class to create instances which represent your data records. These will speed up your develement when CRUD-like tasks need to be done.
To create a Model class, use Sway.data.ActiveRecord.

The basics

To perform a search, a couple of static methods are available. Use the findByX methods to search on a specific field

UserModel.findByUsername('John', function(userRecord) {
     // this === UserModel
}) ;

Of course, the same can be achieved using the more general search method

UserModel.find( {username: 'John'}, function(userRecord) {
     // this === UserModel
}) ;

With find it is also possible to define more fields to search for.
A Model instance, on the otherhand, can be used to create or manipulate data

userRecord = new User() ;                            // create a blank record
userRecord.username = 'John' ;                       // set the username
userRecord.password = 'Secret' ;                     // set the password
userRecord.save(successCallback, errorCallback) ;    // check the result, because this action might fail

Multiple result-sets

In Sway a Model instance can also represent multiple records. Although it always represent a single record, internally this has the whole result set.

its current state will always be a single record, it is possible to navigate from one state to an other

User.search({username: 'John'}, function(record) {   // record is a model instance representing more than on result
      while( record.hasNext() ) {                    // check if there is an other record
           record.next() ;                           // move on record up
           ....
      }
      record.item(1) ;                               // go to second record
      record.prev() ;                                // go to first record. Use 'prev' in combination with 'hasPrev'
}) ;

Advanced

An instance represents one or more records, which depends on how it was created. For example, if a database search returns multiple records, the Model instance represent them all, holding in its current state the first record's values

 User.find({ username: 'John' }, function(ar) {  // ActiveRecord instance, holding multiple records
     console.log("Found " + ar.length + " records) ;
 }) ;

Checkout Sway.data.Model/next:method, Sway.data.Model/prev:method item and hasNext to understand how to deal with multi-record result-sets.

var userRecord = new User({username: 'John', password: 'Secret'}) ;
....
userRecord.save() ;

All fields are accessible as a property of a record

var userRecord = new User() ;
userRecord.username = 'John' ;
userRecord.password = 'Secret' ;

link

()

Defined in src/data/activerecord.js:263

Call this function to make it aware of changes made to the data it relates to. Because a Model instance has no direct link with, for example, a database, this mechanism only works when all changes made to the data are performed by one and the same storage object. This storage object is responsible for the notifications.

Always call unlink to disable this behavior, or when the Model instance otherwise, the If the record or this link is not needed anymore, make sure to remove by calling unlink,
