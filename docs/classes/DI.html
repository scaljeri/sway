DI Class

Defined in: src/di.js:4

Manages classes and/or instances, with each being accessible by a contract. Two types of contracts are available, a contract for instances and classes For registered classes, instances are created when requested (lazy initialization). Also, dependencies are injected (DI) when these instances are created, facilitating loose coupling between classes.

Register a class and setup a contract. ServiceLocator/getDependency:method
will return an instance of this class. Depending on the policy, this class can be used as a singleton.

Parameters:

  • contract String

    name of the contracta

  • type Class

    class name

  • [policy] String optional
    • determines if the class is a singleton (single) or not

Example:

App.locator.registerType("ajax", App.AJAX) ;
App.locator.registerType("util", App.Util, "single") ;
